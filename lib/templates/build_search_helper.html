Select There are there method(Job Name, Build Number,Build Status) to search the build data from the build history on the basis of parse pattern.
If Selection type is Job Name Pass the Job name, The job name should be available in the Build history (If it is not available then do log analysis for that job) the job name will seem like "Performance analysis", 'Upgrade_job" etc (always a string)
If Selection type is Build Number Pass the Build Number, The Build Number should be available in the Build history (If it is not available then do log analysis for that job) the Build Number will seems like 1298, 1222,1,22 etc(always integer)
If Selection type is Build Status Pass the Build Status, The Build Status should be available in the Build history(If it is not available then do log analysis for that job) the Build Status will seems like SUCCESS, FAILED (always string)
